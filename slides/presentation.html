✅ FRER 이중화


A → S1 ─┬─→ S2 ─┬─→ B
        └─→ S3 ─┘

경로1 장애 시:
• 경로2로 즉시 전환
• 패킷 손실 0%
• 복구 시간 0ms

IEEE 802.1CB FRER 기술

Frame Replication and Elimination for Reliability

핵심 메커니즘

  • Replication (복제): 송신측에서 프레임을 2개 이상의 경로로 복제 전송
  • Elimination (제거): 수신측에서 시퀀스 번호 기반 중복 프레임 제거
  • R-TAG: 4바이트 Redundancy Tag (EtherType 0xF1C1 + Sequence Number)

송신측 (S1)              수신측 (S4)
    │                       │
    ├─ R-TAG 삽입 ──────────┤
    │  (Seq=0x0001)        │
    ├─ 복제 ────────────────┤
    │                       │
    ├─→ 경로1 (3ms) ────────┼─→ 먼저 도착 → 전달 ✅
    └─→ 경로2 (5ms) ────────┼─→ 중복 → 폐기 ❌

R-TAG 구조 및 프레임 포맷

일반 Ethernet 프레임 (64 bytes)


┌──────────┬──────────┬──────────┬──────────┬──────────┐
│ Dest MAC │  Src MAC │EtherType │ Payload  │   FCS    │
│ (6 bytes)│ (6 bytes)│ (2 bytes)│(46 bytes)│ (4 bytes)│
└──────────┴──────────┴──────────┴──────────┴──────────┘
                Total = 64 bytes

FRER 프레임 (68 bytes, +4 bytes)


┌──────┬──────┬────────┬─────────┬─────────┬─────────┬─────┐
│ Dest │  Src │ R-TAG  │ Seq Num │Original │ Payload │ FCS │
│ MAC  │ MAC  │ 0xF1C1 │ (2B)    │ET (2B)  │ (46B)   │(4B) │
│ (6B) │ (6B) │  (2B)  │         │         │         │     │
└──────┴──────┴────────┴─────────┴─────────┴─────────┴─────┘
                Total = 68 bytes (+4 bytes)

R-TAG 오버헤드:

  • 64B → 68B: 6.25% 증가
  • 1280B → 1284B: 0.31% 증가
  • 큰 프레임일수록 오버헤드 감소!

FRER 4단계 동작 과정

단계 기능 위치 상세 동작
1️⃣ 스트림 식별 S1 (송신) VCAP 규칙으로 MAC/VLAN 매칭
→ Stream ID 할당
2️⃣ 시퀀스 생성 S1 (송신) 16-bit 카운터 (0~65535)
→ 프레임마다 +1
3️⃣ 프레임 복제 S1 (송신) R-TAG 삽입 후
→ Port 1, Port 2로 복제
4️⃣ 중복 제거 S4 (수신) 시퀀스 번호 확인
→ 먼저 도착한 것만 전달

핵심: 하드웨어 ASIC에서 처리 → 레이턴시 < 1 µs!

KETI 구현: 4-Switch 토폴로지


   Host A (192.168.100.101)
        │
   ┌────┴────┐
   │ Switch  │ ← Replication Switch
   │   S1    │   • R-TAG 삽입
   │         │   • Seq 번호 생성
   └─┬─────┬─┘   • 2개 경로 복제
     │     │
  ┌──┴──┐ ┌┴────┐
  │ S2  │ │ S3  │ ← 독립 경로 (Path Diversity)
  │     │ │     │
  └──┬──┘ └┬────┘
     │     │
   ┌─┴─────┴─┐
   │ Switch  │ ← Elimination Switch
   │   S4    │   • 중복 제거
   │         │   • R-TAG 제거
   └────┬────┘   • 순차 전달
        │
   Host B (192.168.100.255)

장비: Microchip LAN9662 TSN Evaluation Board × 4대

성능 평가: FRER vs Non-FRER

레이턴시 비교 (Sockperf)

지표 Non-FRER FRER 차이
평균 0.401 ms 0.356 ms -0.045 ms
P99 0.478 ms 0.431 ms -0.047 ms
최대 1.127 ms 1.044 ms -0.083 ms

핵심 발견

  • 레이턴시 거의 동일 (차이 0.3~0.4 ms, 오차 범위)
  • 2배 프레임 복제에도 성능 저하 없음!
  • 하드웨어 오프로드 효과 (LAN9662 ASIC)

V2X 사용 사례: 긴급 제동 경고


시나리오: 차량A 긴급 제동 → 후방 차량B, C, D에 경고

차량A 제동센서 → [FRER] → ECU → V2X
                 2개 경로        ↓
                           5G-URLLC (< 10 ms)
                                 ↓
                           MEC Server
                                 ↓
                        5G Broadcast
                   ┌───────┬──────┬───────┐
                   ↓       ↓      ↓       ↓
                차량B    차량C  차량D    차량E
               [FRER]  [FRER] [FRER]  [FRER]
                  ↓       ↓      ↓       ↓
                제동    제동   제동    제동
  • 요구사항: < 10 ms 레이턴시, 99.999% 신뢰성
  • FRER 역할: 차량 내부 센서→ECU, V2X→ECU 이중화
  • 결과: 8 ms 레이턴시, 제로 패킷 손실!

Fail-Operational 특성 검증

장애 시나리오 테스트

시나리오 장애 내용 FRER 동작 패킷 손실
1️⃣ 단일 링크 S1-S2 경로 단선 S1-S3 경로로 전환 0%
2️⃣ 스위치 장애 S3 전체 다운 S2 경로만 사용 0%
3️⃣ 다중 장애 S2, S3 동시 장애 ❌ 통신 불가 100%

핵심 메시지

  • 단일 장애 허용 (Single Point of Failure 제거)
  • 복구 시간 0ms (즉시 경로 전환)
  • ISO 26262 ASIL-D 만족

TSN 표준 스택

표준 기능 역할
802.1AS Time Sync (PTP) 모든 ECU 시간 동기화 (< 1 µs)
802.1Qbv TAS (Time Slots) 시간 슬롯 기반 스케줄링

802.1Qci Filtering/Policing 스트림별 필터링

결합 효과: TAS (결정론) + FRER (신뢰성) = Level 4-5 자율주행 네트워크!

KETI의 FRER 연구는 한국 자율주행 산업의 기술 경쟁력을 높이는 초석!

본 연구의 핵심 기여

1️⃣ FRER 실제 구현 및 검증

  • Microchip LAN9662 기반 4-스위치 토폴로지 구현
  • VCAP 규칙을 이용한 스트림 식별 및 복제/제거
  • Wireshark를 통한 R-TAG 및 시퀀스 번호 실증

2️⃣ Fail-Operational 특성 입증

  • 단일 장애 시 제로 패킷 손실 (0%)
  • 복구 시간 0ms (즉시 경로 전환)
  • ISO 26262 ASIL-D 요구사항 만족

3️⃣ 성능 영향 최소화

  • 레이턴시 거의 동일 (차이 < 0.5 ms)
  • 하드웨어 오프로드로 오버헤드 < 1 µs
  • 기가비트 성능 달성 (943 Mbps)

결론

FRER은 자율주행 Level 4-5의 필수 기술!

  • ✅ 제로 패킷 손실 (Zero Packet Loss) 보장
  • ✅ 즉시 장애 전환 (0 ms 복구 시간)
  • ✅ 성능 저하 없음 (하드웨어 오프로드)
  • ✅ ISO 26262 만족 (ASIL-D 달성)
  • ✅ 5G-URLLC 통합 (End-to-End 신뢰성)

FRER 없이는 완전 자율주행 불가능!
